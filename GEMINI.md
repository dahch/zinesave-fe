# ZineSave App - Project Context

This document provides essential information for AI agents and developers working on the ZineSave App codebase.

## Project Overview
ZineSave is a modern web application that converts web articles into optimized formats (like ePub) for reading on devices like Kindle and Kobo. It features direct integration with cloud storage services (Google Drive, Dropbox, OneDrive) for seamless document management.

- **Primary URL**: https://zinesave.io
- **Backend**: FastAPI (Python) service (communicates via `NEXT_PUBLIC_API_URL`).

## Architecture & Tech Stack
- **Framework**: [Next.js 16](https://nextjs.org) (App Router)
- **Library**: [React 19](https://react.dev)
- **Styling**: [Tailwind CSS 3](https://tailwindcss.com) with custom brand colors.
- **State Management**: [Zustand](https://github.com/pmndrs/zustand) (with persistence).
- **Data Fetching**: [TanStack Query v5](https://tanstack.com/query/latest) & [Axios](https://axios-http.com).
- **Internationalization**: [react-i18next](https://react.i18next.com) (Supports English and Spanish).
- **Icons**: [Lucide React](https://lucide.dev).
- **Feedback**: [Sonner](https://sonner.emilkowal.ski) for toast notifications.

## Project Structure
- `src/app/`: Next.js pages, layouts, and route groups (e.g., `(auth)`).
- `src/components/`: Reusable UI components.
- `src/lib/`: Core utilities, API client configuration, and i18n setup.
- `src/store/`: Global state management using Zustand (e.g., `auth.ts`).
- `src/types/`: TypeScript interfaces and type definitions.
- `src/locales/`: Translation files (JSON) for English (`en.json`) and Spanish (`es.json`).

## Core Conventions
- **API Communication**: Always use the central axios instance from `@/lib/api`. It includes interceptors for Bearer token authentication and automatic logout on 401 errors.
- **Authentication**: Managed via `useAuthStore` in `@/store/auth`. Tokens are persisted in local storage.
- **Internationalization**: Use the `useTranslation` hook for all user-facing text. Keys are nested in `src/locales/`.
- **Styling**: Use Tailwind utility classes. Prefer brand colors: `brand-navy`, `brand-orange`, and `brand-light`.
- **Data Fetching**: Use TanStack Query hooks for managing server state. Invalidate queries (e.g., `dashboard-home`) after mutations to ensure data consistency.

## Building and Running
The project uses `pnpm` as the package manager.

- **Development**: `pnpm dev` (starts the development server with Turbo).
- **Production Build**: `pnpm build`.
- **Production Start**: `pnpm start`.
- **Linting**: `pnpm lint`.

## Environment Variables
The application requires the following environment variables:
- `NEXT_PUBLIC_API_URL`: Base URL for the FastAPI backend.
- `NEXT_PUBLIC_APP_URL`: Base URL for the frontend (used in SEO metadata).
